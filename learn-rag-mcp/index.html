How the pieces fit together

AI → LLM → {RAG, Agents, MCP}: the LLM is the engine; the three tools below extend it.

LLMs are the brain. RAG feeds them facts they were never trained on. Agents let them take actions. MCP is the universal standard for plugging tools and data into them.

What you'll be able to do by the end

Understand

  • Explain LLMs, tokens & context windows plainly
  • Describe exactly how RAG works and why it beats fine-tuning for facts
  • Tell the difference between an LLM, an agent, and a tool
  • Explain what MCP solves and why it matters

Build

  • A minimal RAG pipeline from scratch in Python
  • A PDF Q&A app you can ask questions to
  • A working MCP server with custom tools
  • A fully local EDA (data-analysis) agent with Ollama

What you'll need (for the build pages)

  • Python 3.10+ installed
  • A terminal / command prompt
  • An API key (OpenAI or Anthropic) — or Ollama for free/local
  • ~30 minutes per build
  • A PDF (Part 06) and a CSV (Part 07) to test with
  • Curiosity — that's the main one 🙂
