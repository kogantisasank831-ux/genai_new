⚙️ Build: A Simple RAG App

Time to make it real. We'll build a working RAG system in ~60 lines of Python — no frameworks, so you see every moving part from Part 02 with your own eyes.

⏱ ~60 lines of Python Difficulty · Beginner Prereq · Part 02 (RAG)
💡
What we're building

A tiny knowledge base of facts about a fictional company. You ask a question in plain English; the program finds the most relevant facts and the LLM answers using only those. By the end you'll have run real RAG end-to-end.

The whole app, at a glance

Every RAG app is the same pipeline. We'll build it left to right: index your documents once, then for each question retrieve the closest chunks and let the LLM answer using only them.

The simple RAG app: a one-time index pass (top), then a per-question retrieve-and-generate pass (bottom) that reads from the same index.

3 · Step 1 — Embed the chunks (indexing)

We turn each chunk into a vector once, up front. We store them alongside the original text so we can return the text after a match.

STEP 1 · INDEX

Embed every document and keep (text, vector) pairs — that list is your search index.

python
import os, numpy as np
from openai import OpenAI

client = OpenAI()  # reads OPENAI_API_KEY from the environment

def embed(text):
    """Turn text into a vector (list of floats)."""
    r = client.embeddings.create(model="text-embedding-3-small", input=text)
    return np.array(r.data[0].embedding)

# Build the "index": a list of (text, vector) pairs
index = [(doc, embed(doc)) for doc in documents]
print(f"Indexed {len(index)} chunks.")

4 · Step 2 — Retrieve relevant chunks

Given a question, embed it and find the chunks whose vectors are closest. We use cosine similarity — the standard meaning-distance measure from Part 02.

STEP 2 · RETRIEVE

Embed the question, score every chunk by cosine similarity, sort, and take the top-k.

python
def cosine(a, b):
    """1.0 = identical meaning, 0 = unrelated."""
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

def retrieve(question, top_k=2):
    q_vec = embed(question)
    # score every chunk, sort by similarity, take the best
    scored = [(cosine(q_vec, vec), text) for text, vec in index]
    scored.sort(reverse=True)
    return [text for _, text in scored[:top_k]]
✅
That's the entire "vector database" in 3 lines

A real vector DB (Chroma, FAISS, Pinecone) does exactly this — score + sort + top-k — but optimized to handle millions of vectors fast. Here you see the actual mechanism.

5 · Step 3 — Generate the answer

We stuff the retrieved chunks into a prompt (exactly like the example in Part 02) and ask the LLM to answer using only them.

STEP 3 · GENERATE

Build a prompt from the retrieved chunks, instruct the model to use only that context, and return its answer.

python
def answer(question):
    chunks = retrieve(question)
    context = "\n".join(f"- {c}" for c in chunks)

    prompt = f"""Answer the question using ONLY the context below.
If the answer isn't there, say you don't know.

Context:
{context}

Question: {question}"""

    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}],
    )
    return resp.choices[0].message.content, chunks

6 · The complete program

Here's everything in one file. Copy it into app.py and you're done.

python
# app.py — complete
import os, numpy as np
from openai import OpenAI

client = OpenAI()

documents = [
    "Acme Inc was founded in 2010 and is headquartered in Berlin.",
    "Employees get 25 vacation days per year, accrued monthly.",
    "The company's flagship product is a project tool called BoltBoard.",
    "Remote work is allowed up to 3 days per week.",
    "The support team can be reached at help@acme.example.",
]

def embed(text):
    r = client.embeddings.create(model="text-embedding-3-small", input=text)
    return np.array(r.data[0].embedding)

def cosine(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# --- INDEX once ---
index = [(doc, embed(doc)) for doc in documents]

def retrieve(question, top_k=2):
    q = embed(question)
    scored = sorted(((cosine(q, v), t) for t, v in index), reverse=True)
    return [t for _, t in scored[:top_k]]

def answer(question):
    chunks = retrieve(question)
    context = "\n".join(f"- {c}" for c in chunks)
    prompt = f"Answer using ONLY this context. If absent, say you don't know.\n\n" \
             f"Context:\n{context}\n\nQuestion: {question}"
    resp = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}])
    return resp.choices[0].message.content, chunks

# --- Interactive loop ---
if __name__ == "__main__":
    print("Ask me about Acme Inc (Ctrl+C to quit)\n")
    while True:
        q = input("❓ ")
        ans, used = answer(q)
        print(f"\n💡 {ans}\n")
        print(f"   (retrieved: {used})\n")
