Part 01 · Foundation

LLMs: The Foundation

Before RAG, agents, or MCP make any sense, you need a solid mental model of what a Large Language Model is — and, crucially, what it can't do on its own.

⏱ ~15 min read Difficulty · Beginner Prereq · none

1 · What is an LLM, really?

An LLM (Large Language Model) is a program trained on a massive amount of text — books, websites, code, articles — until it becomes extremely good at one deceptively simple task: predicting what text comes next.

That's it. ChatGPT, Claude, Gemini, Llama — under the hood, they are all giant "autocomplete" engines. The magic is that when you make autocomplete big enough and train it on enough text, it starts to look like reasoning, writing, translating, and answering questions.

💡
Analogy

Imagine someone who has read nearly everything ever written, but has a strange memory: they remember patterns of how words and ideas fit together, not the exact source. Ask them anything and they'll give a fluent, plausible answer drawn from those patterns. That's an LLM — brilliant at language, but with no live access to facts beyond what it absorbed during training.

2 · The core trick: predicting the next token

When you send text to an LLM, it doesn't "look up" an answer. It calculates, for every possible next word, a probability — and picks one. Then it adds that word to the input and repeats. One word at a time.

Generation is a loop: predict one token, append it, predict again — until the model decides to stop.

This is why an LLM can produce a confident answer that is completely wrong — it's optimizing for "plausible-sounding next word," not "verified truth." This behavior is called hallucination, and fixing it is exactly what RAG (Part 02) is for.

3 · Tokens & the context window

LLMs don't read whole words or letters — they read tokens. A token is a chunk of text, roughly ¾ of a word on average. "Understanding" might be one token; "antidisestablishmentarianism" might be five.

The context window is the maximum number of tokens the model can "see" at once — your prompt plus its reply must fit inside it. Think of it as the model's short-term memory / desk space.

Key insight that motivates RAG

The context window is limited. Even large windows can't hold your entire company wiki, every PDF you own, or a live database. So instead of stuffing everything in, we'll learn to retrieve only the relevant pieces and put just those into the prompt. That's RAG in one sentence.

5 · Calling an LLM — your first code

Everything we build later starts with a single API call. Here's the simplest possible version with two popular providers. You'll need an API key (free credits are usually available on signup).

Option A — Anthropic Claude

python
# pip install anthropic
import anthropic

client = anthropic.Anthropic(api_key="YOUR_KEY")

msg = client.messages.create(
    model="claude-opus-4-8",        # latest, most capable
    max_tokens=300,
    messages=[
        {"role": "user", "content": "Explain RAG in one sentence."}
    ],
)
print(msg.content[0].text)

Option B — OpenAI

python
# pip install openai
from openai import OpenAI

client = OpenAI(api_key="YOUR_KEY")

resp = client.chat.completions.create(
    model="gpt-4o",
    messages=[
        {"role": "user", "content": "Explain RAG in one sentence."}
    ],
)
print(resp.choices[0].message.content)
🧩
The "messages" pattern

Notice both use a list of messages with role + content. Roles are usually system (instructions/persona), user (the human), and assistant (the model's past replies). This conversation format is the backbone of every chat app, agent, and RAG system you'll build.

⚠️
Never hard-code keys in real projects

Use an environment variable instead: import os → api_key=os.environ["ANTHROPIC_API_KEY"], and set it in your shell with export ANTHROPIC_API_KEY=... (macOS/Linux) or setx ANTHROPIC_API_KEY ... (Windows).

6 · Why this leads to RAG, Agents & MCP

You now have the foundation. The rest of this guide is about overcoming the three limitations:

  • The LLM doesn't know your data → we'll retrieve it and inject it (RAG).
  • The LLM can't act → we'll give it tools and a loop (Agents).
  • Wiring tools is repetitive → we'll use a standard protocol (MCP).

Then we'll put it all together and build two real apps.

🎯
One-line summary to remember

An LLM is a next-token predictor with frozen, general knowledge and no ability to act. Everything else in this guide is about feeding it the right context and connecting it to the real world.
