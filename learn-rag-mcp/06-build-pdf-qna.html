3 · Step 1 — Load & extract PDF text

PDFs store text page-by-page. We pull it out and keep the page number so we can cite sources later.

python
# STEP 1 · LOAD
from pypdf import PdfReader

def load_pdf(path):
    """Return a list of (page_number, page_text)."""
    reader = PdfReader(path)
    pages = []
    for i, page in enumerate(reader.pages):
        text = page.extract_text() or ""
        if text.strip():
            pages.append((i + 1, text))
    return pages

4 · Step 2 — Chunk the text

A page can be too long for precise retrieval, so we split each page into overlapping chunks (the rules from Part 02). We carry the page number along as metadata.

python
# STEP 2 · CHUNK
def chunk_text(text, size=800, overlap=150):
    """Split text into overlapping windows of characters."""
    chunks, start = [], 0
    while start < len(text):
        end = start + size
        chunks.append(text[start:end])
        start = end - overlap          # step back to overlap
    return chunks

def build_chunks(pages):
    """Flatten pages into chunks with page metadata."""
    out = []
    for page_no, text in pages:
        for j, ch in enumerate(chunk_text(text)):
            out.append({"id": f"p{page_no}-c{j}",
                        "text": ch,
                        "page": page_no})
    return out

5 · Step 3 — Embed & store in Chroma

We create a Chroma collection that uses OpenAI for embeddings, then add all chunks. Chroma embeds and stores them; this is the one-time indexing phase.

python
# STEP 3 · INDEX
import os, chromadb
from chromadb.utils import embedding_functions

def get_collection():
    client = chromadb.PersistentClient(path="./pdf_db")   # saves to disk
    openai_ef = embedding_functions.OpenAIEmbeddingFunction(
        api_key=os.environ["OPENAI_API_KEY"],
        model_name="text-embedding-3-small")
    return client.get_or_create_collection(
        name="pdf", embedding_function=openai_ef)

def index_pdf(path):
    col = get_collection()
    chunks = build_chunks(load_pdf(path))
    col.add(
        ids=[c["id"] for c in chunks],
        documents=[c["text"] for c in chunks],
        metadatas=[{"page": c["page"]} for c in chunks])
    print(f"Indexed {len(chunks)} chunks from {path}")
    return col
💡
Why metadata matters

We attach {"page": n} to each chunk. After retrieval, Chroma hands it back, so we can tell the user "answer found on page 7." That's how citations work.

6 · Step 4 — Retrieve & answer

Query the collection for the top chunks, build the grounded prompt, and call the chat model. Note we ask Chroma to return metadata so we can cite pages.

python
# STEP 4 · ANSWER
from openai import OpenAI
oa = OpenAI()

def ask(col, question, top_k=4):
    res = col.query(query_texts=[question], n_results=top_k)
    chunks = res["documents"][0]
    pages  = [m["page"] for m in res["metadatas"][0]]

    context = "\n\n".join(
        f"[page {p}] {c}" for c, p in zip(chunks, pages))

    prompt = f"""You are a precise document assistant. Answer the
question using ONLY the context. Cite the page numbers you used.
If the answer isn't in the context, say you don't know.

Context:
{context}

Question: {question}"""

    resp = oa.chat.completions.create(
        model="gpt-4o-mini",
        messages=[{"role": "user", "content": prompt}])
    return resp.choices[0].message.content, sorted(set(pages))

7 · The complete CLI app

Save as pdf_qna.py. Run with python pdf_qna.py mydoc.pdf. It indexes once (skips re-indexing if the DB already exists), then chats.

python · pdf_qna.py — complete
import os, sys, chromadb
from pypdf import PdfReader
from chromadb.utils import embedding_functions
from openai import OpenAI

oa = OpenAI()

def load_pdf(path):
    reader = PdfReader(path)
    return [(i+1, p.extract_text() or "")
            for i, p in enumerate(reader.pages)]

def chunk_text(t, size=800, overlap=150):
    out, s = [], 0
    while s < len(t):
        out.append(t[s:s+size]); s += size - overlap
    return [c for c in out if c.strip()]

def get_collection():
    client = chromadb.PersistentClient(path="./pdf_db")
    ef = embedding_functions.OpenAIEmbeddingFunction(
        api_key=os.environ["OPENAI_API_KEY"],
        model_name="text-embedding-3-small")
    return client.get_or_create_collection("pdf", embedding_function=ef)

def index_pdf(col, path):
    ids, docs, metas = [], [], []
    for page_no, text in load_pdf(path):
        for j, ch in enumerate(chunk_text(text)):
            ids.append(f"p{page_no}-c{j}"); docs.append(ch)
            metas.append({"page": page_no})
    col.add(ids=ids, documents=docs, metadatas=metas)
    print(f"✅ Indexed {len(ids)} chunks.")

def ask(col, question, top_k=4):
    r = col.query(query_texts=[question], n_results=top_k)
    chunks, metas = r["documents"][0], r["metadatas"][0]
    pages = [m["page"] for m in metas]
    context = "\n\n".join(f"[page {p}] {c}" for c,p in zip(chunks,pages))
    prompt = ("Answer using ONLY the context and cite page numbers. "
              "If absent, say you don't know.\n\nContext:\n"
              + context + f"\n\nQuestion: {question}")
    resp = oa.chat.completions.create(model="gpt-4o-mini",
        messages=[{"role":"user","content":prompt}])
    return resp.choices[0].message.content, sorted(set(pages))

if __name__ == "__main__":
    col = get_collection()
    if col.count() == 0:                 # only index the first time
        if len(sys.argv) < 2:
            print("Usage: python pdf_qna.py <file.pdf>"); sys.exit()
        index_pdf(col, sys.argv[1])
    print("Ask about your PDF (Ctrl+C to quit)\n")
    while True:
        q = input("❓ ")
        ans, pages = ask(col, q)
        print(f"\n💡 {ans}\n📄 sources: pages {pages}\n")
✅
Re-indexing

The app skips indexing if ./pdf_db already has data. To index a new PDF, delete the pdf_db folder (or use a different collection name) and run again.

8 · Step 5 — Add a web UI (Streamlit)

A terminal is fine for testing, but a real app needs a UI. Streamlit turns a Python script into a web app with almost no extra code — including a file uploader.

bash
pip install streamlit
streamlit run app_ui.py
python · app_ui.py
import streamlit as st
from pdf_qna import get_collection, index_pdf, ask   # reuse our functions!

st.title("📄 Chat with your PDF")

uploaded = st.file_uploader("Upload a PDF", type="pdf")
if uploaded:
    with open("uploaded.pdf", "wb") as f:
        f.write(uploaded.read())
    col = get_collection()
    if col.count() == 0:
        with st.spinner("Indexing..."):
            index_pdf(col, "uploaded.pdf")

    question = st.text_input("Ask a question about the document:")
    if question:
        with st.spinner("Thinking..."):
            answer, pages = ask(col, question)
        st.markdown(f"**Answer:** {answer}")
        st.caption(f"📄 Sources: pages {pages}")
🧩
Notice the payoff of clean functions

The UI just imports and reuses get_collection, index_pdf, and ask from the CLI app. Same logic, new front-end. This is why we kept the RAG logic in tidy functions.

Roughly what your Streamlit app looks like — upload, ask, get a cited answer.

9 · Step 6 — Expose it as an MCP server (full circle)

Remember Part 04? Let's turn this RAG app into a reusable MCP tool so Claude Desktop (or any agent) can query your PDF. It's just a thin wrapper:

python · pdf_mcp.py
from mcp.server.fastmcp import FastMCP
from pdf_qna import get_collection, ask

mcp = FastMCP("pdf-knowledge")
col = get_collection()      # assumes the PDF is already indexed

@mcp.tool()
def search_pdf(question: str) -> str:
    """Answer a question using the indexed PDF document."""
    answer, pages = ask(col, question)
    return f"{answer}\n\n(Sources: pages {pages})"

if __name__ == "__main__":
    mcp.run()
🎉
You've now connected every concept in this guide

An LLM answers using RAG over your PDF, packaged as an agent tool via the MCP standard. Add it to Claude Desktop's config (Part 04, §7) and ask Claude about your document from inside the app.

10 · Troubleshooting & tips

SymptomCause & fix
Empty / garbled answersPDF is scanned images, not text. pypdf can't read those — run OCR first (e.g. ocrmypdf) or use a vision model.
"I don't know" too oftenIncrease top_k (try 6–8), or reduce chunk size for finer retrieval.
Wrong/irrelevant chunksChunks too big — lower size to ~500. Add overlap so facts aren't split.
Re-indexes every runThat's expected if you delete pdf_db. Keep the folder to reuse the index.
Slow / costlyIndex once and persist (we do). Use text-embedding-3-small (cheap) and a small chat model.
Want it fully local/freeSwap embeddings → sentence-transformers; swap chat → Ollama (llama3). Chroma works with both.

Production hardening checklist

  • Store API keys in environment variables / a secrets manager — never in code
  • Handle multiple documents (add a source field to metadata; filter by it)
  • Add conversation memory so follow-up questions keep context
  • Show the actual retrieved snippets so users can verify the answer
  • Add rate limiting and error handling around API calls
  • Evaluate quality on real questions before shipping (does it cite correctly?)
