# Drawer, reading chrome, theme, and output-guardrail update

- Added an accessible navigation resizer for desktop and the open mobile drawer. The previous drawer width is the minimum; users can drag wider, shrink back, use arrow keys, or double-click to reset. Width persists locally.
- Removed the Focus and Reader controls from the top bar and retired their saved settings.
- Made the wide reading layout the default.
- Changed first-visit theme selection to light while preserving an explicit user theme choice through the existing toggle.
- Updated Foundations of LLMs Figure 3 so post-generation validation explicitly includes output guardrails, with a note distinguishing synchronous guardrails from evaluation.
