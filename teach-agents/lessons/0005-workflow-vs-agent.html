Lesson 5 · Concept

Workflow or agent? Choosing per scenario

"Agentic" is not all-or-nothing. The real question is how much of the decision-making you hand to the LLM at runtime — and most reliable production "agents" are actually workflows with one or two genuinely autonomous steps.

⏱ ~20 min Type · Concept + decision method Prereq · Lessons 1–4 Next · Lesson 6 (capstone)
🎯
By the end of this lesson you will be able to
  • Place any system on the spectrum from fixed workflow to autonomous agent.
  • Apply a 4-question decision rule to choose the right shape for an EnM scenario.
  • Explain why "the simplest thing that works" is usually a workflow, not full autonomy.

1 · The spectrum, not a binary

"Agentic" is not all-or-nothing. The real question is: how much of the decision-making do you hand to the LLM at runtime? There's a whole spectrum:

Fixed codeno LLM
WorkflowLLM in fixed steps
Workflow + local autonomyLLM chooses within a step
Autonomous agentLLM chooses everything

More autonomy to the right. More predictability, lower cost, easier debugging to the left.

Lesson 4's orchestrator sits at the "Workflow" point: the model makes decisions inside each agent (which tag to inspect, how to phrase a diagnosis), but the order of agents is fixed by us. We could push right — let an LLM decide which agent runs next — but, as you'll see, that's rarely the right move.

2 · Workflow vs. autonomous, compared

📘
Workflow

A system where you define the sequence of steps in advance. The path is the same every run. LLMs may do work within steps, but they don't choose the path.

📘
Autonomous agent

A system where the LLM decides, at runtime, what to do next and in what order. The path varies from run to run depending on what it discovers.

A workflow runs a predetermined sequence A→B→C; an autonomous agent loops, deciding its own next step each time until it chooses to finish.
WorkflowAutonomous agent
Who decides the steps?You, in advance (fixed path)The LLM, at runtime
PredictabilityHigh — same path every timeLower — path varies

3 · The default that surprises people

🔑
Prefer the simplest thing that works

Hard-won industry experience (and Anthropic's guidance): most reliable "agents" in production are actually workflows with one or two genuinely autonomous steps. Full autonomy is a cost — in latency, money, and unpredictability — that you pay only where the path truly cannot be known in advance.

This runs against the hype, which sells maximum autonomy as maximum sophistication. In reality, the sophisticated move is to localize autonomy: let the LLM be creative exactly where creativity is needed, and pin everything else down so it's reliable. A system that does the right five steps every time is worth more than one that might do the right steps, or might not.

✅
In plain English

Don't give the AI the steering wheel for the whole journey when you only need it to parallel-park. Fixed route, AI for the tricky bit.

4 · A 4-question decision rule

For any scenario in the spreadsheet, ask these in order. The first "yes" usually decides the shape.

Are the steps the same every time?

If yes → workflow. The sequence is known, so encode it. (EnM #2 reliability chain; #8 margin chain.)

Does the next step depend on what the last step found?

If yes, you need some autonomy — but localize it. Let an agent choose within a step, not redesign the whole flow. (E.g. the Anomaly Agent choosing which tags to drill into.)

Is the task genuinely open-ended exploration?

Only then reach for a fully autonomous orchestrator. (EnM #6 EDA — "suggest nuances … candidates for digital intervention" — is genuinely open-ended: you don't know in advance what you'll find.)

Where's the risk?

Anything touching live control, safety, or money (EnM #1 console, #8 margins) wants a workflow with explicit constraint/guardrail agents — never free roaming. A wandering autonomous agent near a live reactor is unacceptable regardless of how clever it is.

The 4-question decision guide — walk the questions in order; the first "yes" usually decides the shape, and risk always pulls you back toward a guarded workflow.

5 · Applying it to the "think-through" rows

The spreadsheet explicitly flags #3 (Energy & Carbon) and #5 (Planning & Economics) as "need to think through how to break it up into agents." The decision rule gives you a defensible starting decomposition for each. Treat these as proposals to argue about, not final answers — which is exactly what the spreadsheet is asking for.

EnM #3 — Energy & Carbon Optimization ("Digital Energy Manager")
Shape: workflow (steps are stable; touches energy cost & compliance → rule #1 and #4).

1. Measurement Agent — a tool that pulls energy use & emissions per unit. (Classical data access.)
2. Forecast Agent — projects demand / load ahead. (Classical forecasting, like Lesson 3.)
3. Optimization Agent — a solver (the differential-evolution "Evolution Agent" pattern) that finds the lowest-carbon feasible setpoints. (Classical optimization.)
4. Constraint Agent — enforces emissions/compliance limits on any proposed setpoint. (Guardrail.)
5. Explainer (LLM) — turns the numbers into guidance for the energy manager.

Autonomy is near zero; the LLM only contextualizes and explains. The intelligence is in the solver.
EnM #5 — Planning & Economics ("Digital Economist")
Shape: workflow with a human in the loop (money at stake → rule #4 dominates).

1. Data Agent — gathers prices, margins, constraints. (Classical data access.)
2. Scenario Agent — generates a handful of plan options. (Here a little LLM creativity is useful — this is the one place autonomy earns its keep.)
3. Economics Agent — scores each option. (The solver / economic model.)
4. Present ranked options to a human planner — who makes the final call.

Autonomy is confined to generating scenarios; the choice stays bounded and human-reviewed because the consequences are financial. Never let an autonomous agent commit money unsupervised.

🎯
Recap
  • Autonomy is a spectrum, not a switch — measured by how much the LLM decides at runtime.
  • Workflow = you fix the steps; autonomous = the LLM picks the steps.
  • Prefer the simplest thing that works: usually a workflow with localized autonomy.
  • Decision rule: same steps? → workflow. Step depends on findings? → localize autonomy. Open-ended? → autonomous. Risky/money/safety? → workflow + guardrails.
  • EnM #3 and #5 are best modeled as workflows; the LLM explains, the solver decides.
