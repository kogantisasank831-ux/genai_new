Lesson 02 · Reliability layer

Build the Shared LLM Client

One carefully designed wrapper turns six prompts into a maintainable agent subsystem.

⏱ Study + implementation · 4–6 hIntermediate · PythonBased on the uploaded ATS app

1 · Why every agent should share one client

Without a wrapper, six agents become six slightly different implementations of timeouts, parsing, validation and error handling. Differences become bugs. The uploaded app uses call_claude_json() as the single choke point.

Consistency

One retry policy, one JSON rule and one error shape.

Observability

Every call logs tokens, latency, status and entity context.

Replaceability

Provider-specific behavior stays out of the recruitment agents.

The wrapper turns unreliable free-form generation into a typed application dependency. Repair and retry paths are deliberately bounded.

2 · Start with the output contract

Define what the application needs before writing the prompt. This makes invalid states visible and gives tests something concrete to assert.

schemas/screening.py
from pydantic import BaseModel, Field

class ResumeScreeningResult(BaseModel):
    match_score: float = Field(ge=0, le=100)
    experience_fit: str
    missing_essential_skills: list[str] = []
    strengths: list[str] = []
    risks: list[str] = []
    recommendation: str
    rationale: str
✅
Strong contract rule

Use numeric bounds, literals/enums where drift is unacceptable, short lists and required evidence fields. Do not accept a huge unstructured “analysis” blob and parse it later.

3 · Make one async model call

agents/client_minimal.py
import asyncio, json, time
from anthropic import AsyncAnthropic
from pydantic import BaseModel

JSON_ONLY = "Return one valid JSON object. No prose or markdown."
RETRY_DELAYS = (1, 3)

async def call_llm_json(*, system: str, user: str,
                        output_model: type[BaseModel],
                        model: str, timeout_s: int = 60) -> dict:
    client = AsyncAnthropic(timeout=timeout_s)
    started = time.perf_counter()

    for attempt in range(3):
        try:
            response = await client.messages.create(
                model=model,
                system=f"{system}

{JSON_ONLY}",
                messages=[{"role": "user", "content": user}],
                max_tokens=1500,
            )
            raw = "".join(block.text for block in response.content
                          if block.type == "text")
            data = json.loads(strip_json_fences(raw))
            return output_model.model_validate(data).model_dump()

        except TRANSIENT_ERRORS:
            if attempt == 2:
                raise
            await asyncio.sleep(RETRY_DELAYS[attempt])

    raise RuntimeError("unreachable")

The app keeps the model name in settings, applies a timeout and sends no provider parameter that the configured model rejects. Keeping the model configurable makes upgrades and rollback easier.

4 · Parse defensively, repair only once

agents/json_tools.py
def strip_json_fences(raw: str) -> str:
    text = raw.strip()
    if text.startswith("```"):
        text = text[3:]
        if text[:4].lower() == "json":
            text = text[4:]
        if text.endswith("```"):
            text = text[:-3]
    return text.strip()

async def repair_once(client, model: str, raw: str) -> dict:
    repair = await client.messages.create(
        model=model,
        system="Return one valid JSON object only.",
        messages=[{"role": "user", "content": f"Repair this JSON:
{raw}"}],
        max_tokens=1500,
    )
    fixed = "".join(x.text for x in repair.content if x.type == "text")
    return json.loads(strip_json_fences(fixed))

A repair pass is useful for accidental formatting mistakes. It should not become an endless “ask the model to fix itself” loop. After one repair and bounded retries, fail clearly.

5 · Retry only failures that may recover

Error classRetry?Reason
Timeout / connectionYesA temporary network issue may disappear.
Rate limit / provider 5xxYesBackoff can let capacity recover.
Bad request / invalid model parameterNoThe same request will fail again. Fix code or config.
Pydantic schema failureUsually failSurface a degraded result only if your product explicitly supports it.
⏳
Production improvement

Add random jitter to exponential backoff and honor provider retry headers. This prevents many workers from retrying at exactly the same moment.

6 · Log every run durably

agents/run_log.py
async def log_agent_run(*, name, entity_id, status,
                        input_digest, output, usage, latency_ms, error=None):
    # Use an independent DB session so the trace survives even when
    # the caller's business transaction rolls back.
    async with SessionLocal() as db:
        db.add(AgentRun(
            agent_name=name,
            entity_id=entity_id,
            status=status,
            input_digest=input_digest,   # lengths/hash, not the full CV
            output=output,
            prompt_tokens=usage.input_tokens,
            completion_tokens=usage.output_tokens,
            latency_ms=latency_ms,
            error_detail=error,
        ))
        await db.commit()

The uploaded app logs in a separate database session. That is a subtle and excellent decision: a failed agent run remains observable even when the business transaction is rolled back.

7 · Convert provider failure into product behavior

Agent succeeds

Validate → return to service → persist selected fields → audit.

Agent fails

Log failure → return a typed 502 error → keep manual workflow controls available.

AI should improve throughput, not become a single point of failure. In the ATS, a recruiter can still write a JD, manually review a resume or schedule an interview.

8 · Best-practice upgrades

  • Prefer provider-native structured output when available; keep Pydantic validation anyway.
  • Hash or summarize sensitive inputs in traces rather than logging raw CVs.
  • Add idempotency keys for user-triggered re-runs.
  • Track prompt version and schema version with every run.
  • Use a circuit breaker when a provider is repeatedly failing.
  • Measure cost and quality by agent name, not only globally.
