Build the Interview Agents

See where AI helps panels prepare and synthesize—and where ordinary Python must remain in charge.

1 · The interview intelligence loop

Scheduling is a hybrid AI + deterministic workflow. Questions prepare the panel; summarization carries evidence forward without rewriting the panel decision.

2 · Agent D — interview scheduling

The model is useful for ranking human-friendly options and explaining trade-offs. It is not trusted as the final collision detector.

services/scheduling_simplified.py
def overlaps(start_a, end_a, start_b, end_b) -> bool:
    return start_a < end_b and start_b < end_a

async def suggest_slots(db, request):
    busy = await load_panel_busy_intervals(db, request.panelist_ids)
    result = await interview_scheduling.run(
        candidate_windows=serialize(request.candidate_windows),
        panelists=serialize_busy(busy),
        round=request.round,
    )

    for proposal in result["proposals"]:
        start, end = parse_iso(proposal["start"]), parse_iso(proposal["end"])
        proposal["verified"] = bool(
            start and end and end > start and
            not any(overlaps(start, end, b_start, b_end)
                    for intervals in busy.values()
                    for b_start, b_end in intervals)
        )
    return result
🛡️
The most important pattern in this course

Use AI for judgment; use code for invariants. Date overlap is deterministic, so Python must verify it. Similar examples: permissions, numeric thresholds, duplicate IDs, legal state transitions and required fields.

3 · Agent E — tailored interview questions

The agent generates 8–12 questions for the current round. Technical rounds probe technical depth; the management round shifts to behavioral and leadership evidence.

services/question_generation.py
async def generate_questions(interview, application):
    prior = await load_prior_round_feedback(interview.application_id,
                                            before_round=interview.round)
    focus = collect_weaknesses_and_next_focus(prior)

    result = await interview_questions.run(
        round=interview.round,
        position_title=application.rrf.position_title,
        jd_markdown=latest_jd(application.rrf),
        essential_skills=essential_skills(application.rrf),
        cv_text=application.candidate.cv_text[:12_000],
        prior_round_focus=focus,  # server-controlled, not supplied by browser
    )
    interview.ai_interview_questions = result  # cache for the panel
    return result

Grounded

Each question ties back to a claim in the CV or a requirement in the JD.

Round-aware

R2 goes deeper than R1; management avoids low-level coding.

Continuity-aware

Weaknesses and next-round focus are injected by the server.

4 · Generate preparation asynchronously

services/question_background_task.py
async def generate_questions_in_background(interview_id, user_id):
    # The interview is already committed. AI preparation is best-effort.
    try:
        async with SessionLocal() as db:
            interview = await interview_repo.get_by_id(db, interview_id)
            if not interview or interview.ai_interview_questions is not None:
                return
            await generate_questions(db, user_id, interview_id)
    except Exception:
        logger.warning("question pre-generation failed", exc_info=True)
        # Do not cancel or corrupt the scheduled interview.

The uploaded app pre-generates questions after scheduling, in its own session, and skips work if questions are already cached. A provider outage cannot undo a successfully scheduled interview.

5 · Agent F — feedback summarization

The summarizer combines overall rating, skill ratings, category assessments, raw notes and earlier round summaries. It is instructed to remain neutral and to echo—not invent—the final recommendation.

services/feedback_summary.py
async def summarize_feedback(interview, feedback):
    prior = await load_prior_round_feedback(interview.application_id,
                                            before_round=interview.round)
    return await feedback_summarization.run(
        round=interview.round,
        overall_rating=feedback.overall_rating,
        recommendation=feedback.recommendation,
        skill_ratings=feedback.skill_ratings,
        attributes=feedback.attributes,
        raw_notes=(feedback.raw_notes or "")[:6_000],
        prior_round_summaries=prior,
    )
🔒
Why prior feedback is server-injected

The browser should not choose which earlier feedback the model sees. The service applies application, round and role rules, then injects only authorized prior summaries. This reduces tampering and accidental data leakage.

6 · Best practices demonstrated here

  • Advisory scheduling plus deterministic non-overlap verification.
  • Panel size limited before agent invocation.
  • Prior-round context selected by server-side authorization logic.
  • Question results cached on the interview.
  • Background generation is best-effort and idempotent.
  • Original ratings and recommendation stay as source of truth.
  • Audit record written when generated questions are persisted.
