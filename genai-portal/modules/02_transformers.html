Module 02 · Foundations track

Transformers — the engine inside every LLM

In Module 01 we treated the model as a black box: f(tokens) → next-token distribution. Now we open the box. The transformer is what makes that function possible, and its central mechanism — self-attention — is the single most important idea in modern AI. Understand it once and embeddings, context windows, KV caches, and even RAG retrieval all click into place.

⏱ Study + practice · 14–22 h Difficulty · Intermediate Prereq · Module 01, linear algebra basics

1 · Executive summary

The transformer (Vaswani et al., 2017, "Attention Is All You Need") replaced recurrent networks with a mechanism that lets every token look directly at every other token in a single, fully-parallel step. This solved two problems at once: long-range dependencies (RNNs forget distant context) and training speed (RNNs are inherently sequential; transformers are not).

❓

Why it exists

RNNs/LSTMs processed text one step at a time, couldn't be parallelized on GPUs, and lost long-range context. The transformer fixed all three.

🔧

Problem it solves

"Which earlier words matter for predicting this one?" Self-attention computes that relevance dynamically, for every pair of tokens, in parallel.

🌐

Where it fits

It is the architecture. LLMs are stacked transformer decoder blocks. Embedding models are transformer encoders. Knowing it explains the whole stack.

🔑
The one idea to remember

Attention is a learned, content-based weighted average. Each token builds a query, compares it against every token's key to get relevance weights, then mixes their values by those weights. Everything else — multi-head, positional encoding, residuals — is engineering around that core.

2 · First-principles breakdown

What — attention as a "soft lookup"

Forget matrices for a second. Imagine reading the sentence: "The trophy didn't fit in the suitcase because it was too big." To understand "it", your brain glances back and decides "it" refers to the trophy, not the suitcase. You attended to the relevant earlier word.

Self-attention does exactly this, mechanically. For every word, it asks: "Given what I'm looking for, how relevant is each other word?" — then blends the relevant words' information into a new, context-aware representation of the current word.

🔎
The dictionary analogy (Q, K, V)

Think of a Python dict lookup. You have a query ("what am I looking for?"), you compare it to every key ("what does each entry offer?"), and you retrieve the matching value. Attention is a soft, differentiable version: instead of one exact match, you get a weighted blend of all values, weighted by how well each key matches your query.

Why — parallelism + global context

The genius is that the relevance comparison for all token pairs is just one big matrix multiplication — which GPUs do extremely fast and fully in parallel. Unlike an RNN, the model doesn't walk left-to-right; it sees the whole sequence at once and learns which connections matter. That's why transformers scaled to billions of parameters and trillions of tokens.

How — five stages, top to bottom

Token IDs[101, 7592, 2026, ...]

Final layer norm + projection→ logits over vocabulary
Next-token distribution

3 · Visual architecture

The decoder block (what LLMs stack)

GPT-style LLMs are a stack of identical decoder blocks. Each block has two sub-layers — masked multi-head self-attention and a feed-forward network — each wrapped in a residual connection and layer normalization.

4 · Deep technical explanation

From token to vector: the embedding layer

The first learned layer is an embedding table: a big matrix of shape [vocab_size, d_model]. Token ID 7592 just selects row 7592 — a dense vector (e.g. 768 or 4096 dimensions). These vectors are learned so that semantically related tokens land near each other. (This is the same idea we exploit directly in Module 04.)

Positional encoding: order from a bag of vectors

Attention is permutation-invariant — by itself it treats input as an unordered set, so "dog bites man" and "man bites dog" would look identical. We inject order by adding a positional encoding to each embedding.

SchemeIdeaUsed by
SinusoidalFixed sine/cosine waves of varying frequency added to embeddingsOriginal transformer
Learned absoluteA trainable vector per positionEarly GPT/BERT
RoPE (Rotary)Rotates Q/K by an angle proportional to position — encodes relative distance, extrapolates to longer contextsLlama, Qwen, most modern LLMs
ALiBiAdds a distance-based penalty to attention scoresSome long-context models
💡

RoPE is why modern models extend context. Because it encodes relative position by rotation, you can interpolate/scale the rotations to stretch an 8k model toward 32k+ — the basis of context-extension tricks.

Figure 3 · Conceptual RoPE: position rotates query and key dimensions; relative angle carries relative distance information.

Scaled dot-product attention, precisely

For a sequence packed into matrices, the whole operation is four lines:

attention_math.py
import numpy as np

def softmax(x, axis=-1):
    e = np.exp(x - x.max(axis=axis, keepdims=True))
    return e / e.sum(axis=axis, keepdims=True)

def attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.T / np.sqrt(d_k)        # [seq, seq] relevance matrix
    if mask is not None:
        scores = np.where(mask, scores, -1e9)   # causal mask: can't see future
    weights = softmax(scores)                 # rows sum to 1
    return weights @ V                         # [seq, d_v] blended values

Why divide by √dₖ? Without it, dot products grow large in high dimensions, pushing softmax into a near-one-hot regime where gradients vanish. Scaling keeps the distribution soft and trainable.

The causal mask: why LLMs can't peek ahead

Decoder LLMs are trained to predict the next token, so token i must only attend to tokens ≤ i. A triangular causal mask sets future scores to −∞ before softmax. This is also what makes the KV cache from Module 01 valid: past keys/values never change, so they can be cached.

Figure 4 · The lower triangle is visible; the upper triangle is masked before softmax so next-token training cannot leak the answer.

Multi-head attention: many perspectives at once

One attention computation captures one kind of relationship. Multi-head attention runs h attentions in parallel on lower-dimensional projections, then concatenates. One head might track syntax (subject↔verb), another coreference ("it"↔"trophy"), another long-range topic. The heads are learned, not assigned.

Figure 5 · h heads learn different relationships in parallel, then merge through an output projection.

Feed-forward, residuals, and normalization

  • Feed-forward network (FFN): a per-token MLP that expands to ~4× width, applies a non-linearity (GELU/SwiGLU), and projects back. This is where much of a model's "knowledge" capacity lives — and where Mixture-of-Experts (MoE) models swap in sparse expert FFNs.
  • Residual connections: x + sublayer(x). Gradients flow straight through the +, enabling 100+ layer depth without vanishing gradients.
  • Layer normalization: stabilizes activations. Modern LLMs use pre-norm (normalize before the sub-layer) and often RMSNorm for efficiency.

Tradeoffs & limitations

Strengths

  • Fully parallel training (GPU-friendly)
  • Direct long-range connections
  • Scales predictably with params/data/compute

The quadratic wall

  • Attention is O(n²) in sequence length — the root cause of context limits & cost
  • Memory-hungry (KV cache grows with context)
  • Mitigations: FlashAttention (faster exact attention), sliding-window, grouped-query attention (GQA), linear-attention variants

6 · Practical Python

Minimal — single-head self-attention from scratch

self_attention.py
import torch, torch.nn.functional as F

def self_attention(x, Wq, Wk, Wv, causal=True):
    # x: [seq, d_model]
    Q, K, V = x @ Wq, x @ Wk, x @ Wv
    d_k = Q.shape[-1]
    scores = Q @ K.transpose(-2, -1) / d_k**0.5
    if causal:
        seq = x.shape[0]
        m = torch.triu(torch.ones(seq, seq), diagonal=1).bool()
        scores = scores.masked_fill(m, float("-inf"))
    weights = F.softmax(scores, dim=-1)
    return weights @ V, weights

Intermediate — multi-head attention as a reusable module

mha.py
import torch, torch.nn as nn, torch.nn.functional as F

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, n_heads):
        super().__init__()
        assert d_model % n_heads == 0
        self.h, self.dk = n_heads, d_model // n_heads
        self.qkv = nn.Linear(d_model, 3 * d_model)   # fused projection
        self.out = nn.Linear(d_model, d_model)

    def forward(self, x):              # x: [B, T, d_model]
        B, T, D = x.shape
        q, k, v = self.qkv(x).chunk(3, dim=-1)
        # split heads → [B, h, T, dk]
        q, k, v = [t.view(B, T, self.h, self.dk).transpose(1, 2) for t in (q, k, v)]
        # PyTorch's fused, FlashAttention-backed kernel (causal=True)
        o = F.scaled_dot_product_attention(q, k, v, is_causal=True)
        o = o.transpose(1, 2).reshape(B, T, D)
        return self.out(o)

Production — inspect a real model & visualize attention

inspect_attention.py
# Pull a real pretrained transformer and read its attention maps.
from transformers import AutoTokenizer, AutoModel
import torch

name = "sentence-transformers/all-MiniLM-L6-v2"
tok = AutoTokenizer.from_pretrained(name)
model = AutoModel.from_pretrained(name, attn_implementation="eager",
                                  output_attentions=True).eval()

enc = tok("The trophy did not fit because it was too big", return_tensors="pt")
with torch.no_grad():
    out = model(**enc)

attn = out.attentions[-1][0]            # last layer: [heads, T, T]
tokens = tok.convert_ids_to_tokens(enc["input_ids"][0])
it_idx = tokens.index("it")
avg = attn.mean(0)[it_idx]            # avg over heads: where does "it" look?
for t, w in sorted(zip(tokens, avg.tolist()), key=lambda p: -p[1])[:4]:
    print(f"{t:>10}  {w:.3f}")
🛠️

This output_attentions=True trick is the foundation of interpretability tooling (BertViz, attention rollout). In production you'll more often use the fused scaled_dot_product_attention kernel for speed — it doesn't expose weights, which is the usual exact-speed vs. interpretability tradeoff.

You don't train transformers locally, but every Ollama model exposes transformer-architecture knobs in its Modelfile and metadata. Understanding the architecture explains exactly what these settings do.

Why local models use GQA. Llama/Qwen use Grouped-Query Attention — many query heads share fewer key/value heads. This shrinks the KV cache (the main VRAM cost at long context), which is exactly why you can run a quantized 7B model with a long context on a consumer GPU. We measure this in Module 03.

8 · Industry perspective

🚀 Startups

Almost never implement attention — they consume pretrained transformers via API or Ollama. The architectural knowledge that pays off: choosing context length vs. cost, knowing why long prompts are quadratic, and picking GQA/long-context models for the workload.

🏢 Enterprises

May fine-tune or distill transformers, run inference servers (vLLM, TGI) that exploit FlashAttention + paged KV cache, and care deeply about throughput economics — which are all downstream of the architecture in this module.

9 · Top mistakes engineers make

❌ Thinking attention "understands"

It's a weighted average of learned value vectors. Powerful, but mechanical — not comprehension.

❌ Forgetting the O(n²) cost

Doubling context can quadruple attention compute/memory. This is the real reason long prompts hurt latency and cost.

❌ Ignoring positional encoding limits

Feeding a model far beyond its trained context (without RoPE scaling) degrades quality sharply — it's extrapolating positions it never saw.

❌ Confusing encoder vs. decoder

BERT (encoder, bidirectional) is for embeddings/classification; GPT (decoder, causal) is for generation. Picking the wrong family wastes effort.

❌ Assuming more heads = better

Beyond a point, extra heads add cost without quality. Head count is a tuned hyperparameter, not a dial you crank.

❌ Reimplementing attention by hand in prod

Use fused kernels (scaled_dot_product_attention, FlashAttention). Hand-rolled loops are slow and numerically fragile.

10 · Interview preparation

Beginner  What problem did the transformer solve over RNNs? ›

RNNs processed tokens sequentially (slow, un-parallelizable) and struggled with long-range dependencies due to vanishing gradients. The transformer uses self-attention to connect any two tokens directly in one parallel step, enabling both long-range modeling and GPU-efficient training.

Beginner  In attention, what are Q, K, and V? ›

Three learned linear projections of each token's embedding. Query = what this token is looking for; Key = what each token offers for matching; Value = the information each token contributes. Output = softmax(Q·Kᵀ/√d)·V — a relevance-weighted blend of values.

Intermediate  Why divide attention scores by √dₖ? ›

In high dimensions, dot products have large variance (∝ dₖ), pushing softmax toward a near one-hot distribution where gradients vanish. Scaling by √dₖ normalizes the variance, keeping the softmax soft and the model trainable.

Intermediate  Why do residual connections matter for deep transformers? ›

Each sub-layer computes x + f(x). The identity path lets gradients flow directly to earlier layers (avoiding vanishing gradients) and lets each block learn a refinement rather than a full transformation, making 100+ layer stacks trainable.

Senior  Your model degrades when prompts exceed ~8k tokens. Why, and what are your options? ›

Likely exceeding the trained context length — positional encodings are extrapolating to unseen positions, and attention quality drops (lost-in-the-middle). Options: switch to a longer-context model; apply RoPE scaling/interpolation; reduce input via better retrieval/chunking (Modules 6–7); summarize or compress context; or use sliding-window/long-context architectures. Measure with a needle-in-haystack eval before/after.

Senior  Explain the KV cache in terms of the causal mask. ›

Because of the causal mask, token i's keys/values never depend on future tokens and never change once computed. So during autoregressive generation we cache each token's K and V; generating token t+1 only requires computing its own Q and attending over the cached K/V — turning per-step cost from O(t²) recompute into O(t). The cache size grows linearly with context, which is the dominant inference memory cost (mitigated by GQA/paged attention).

Staff · design  Design an inference-serving layer to maximize throughput for a 70B model. ›

Use a server like vLLM/TGI exploiting architectural facts: continuous batching (interleave requests at token granularity since each step is independent), paged KV cache (non-contiguous KV blocks → near-zero memory fragmentation, more concurrency), FlashAttention kernels, tensor/pipeline parallelism across GPUs for the large FFN/attention matmuls, quantization (e.g. FP8/INT8) to fit weights and speed matmuls, and GQA models to shrink the cache. Add prefix/prompt caching for shared system prompts. Tune the batch/context tradeoff against the O(n²) attention curve and your latency SLO.

11 · Knowledge check

1 · What does the Query vector represent in self-attention?
Query = what this token seeks. It's dot-producted against every token's Key to compute relevance; the resulting weights blend the Values.
2 · Why is a causal mask used in decoder LLMs?
Next-token prediction requires that position i not "see the answer." The triangular mask sets future scores to −∞ before softmax, and is also what makes the KV cache valid.
3 · The main reason long prompts are expensive is that attention is…
Every token attends to every other token, so the score matrix is n×n. Doubling context roughly quadruples attention compute and memory — the architectural root of context limits.
4 · What does multi-head attention buy you over single-head?
Each head operates on a projected subspace and can specialize (syntax, coreference, long-range topic). Concatenating them gives a richer representation than one head could.

12 · Active recall

Answer first, then reveal. Schedule a re-test tomorrow and next week.

Q State the scaled dot-product attention formula. ›
Attention(Q,K,V) = softmax(QKᵀ / √dₖ) · V — relevance scores, scaled, softmaxed into weights, used to blend values.
Q Why is positional encoding necessary at all? ›
Self-attention is permutation-invariant — without position info it treats the input as an unordered set, so word order would be lost. Positional encodings (sinusoidal, learned, RoPE) inject order.
Q Name the two sub-layers in a decoder block and what wraps each. ›
(1) Masked multi-head self-attention, (2) position-wise feed-forward network. Each is wrapped in a residual connection + layer normalization (modern models use pre-norm / RMSNorm).
Q What is the difference between an encoder and a decoder transformer? ›
Encoders use bidirectional attention (each token sees all tokens) — good for embeddings/understanding (BERT). Decoders use causal/masked attention (each token sees only the past) — good for generation (GPT/Llama).
Q What is GQA and why does it matter for local inference? ›
Grouped-Query Attention shares K/V heads across multiple query heads, shrinking the KV cache. Smaller cache = less VRAM at long context, enabling bigger models / longer contexts on consumer hardware.
