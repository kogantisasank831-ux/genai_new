Synchronous vs Asynchronous Python

Build a precise mental model of synchronous and asynchronous Python so you can write correct code and explain execution choices under interview pressure.

Study + timed coding · 4–6 hDifficulty - Foundation to IntermediatePython - AsyncIO - Concurrency
🎯
Interview outcome

Predict sync and async execution, choose threads, processes, or AsyncIO correctly, find blocking calls, and discuss cancellation, timeouts, backpressure, and the GIL.

1 - The mental model

ModelWhat happensBest fit
SynchronousOne call completes before the caller continues.Simple code, CPU steps, blocking libraries, low concurrency.
AsyncIOOne event-loop thread switches tasks at await points while I/O is pending.Many concurrent network/database/socket waits.
ThreadsOS threads can overlap blocking I/O; Python bytecode CPU work is constrained by the GIL in normal CPython.Blocking libraries and modest I/O concurrency.
ProcessesSeparate interpreters and memory can run CPU work in parallel.CPU-heavy pure Python, isolation, worker pools.
execution.py
import asyncio
import time

async def io_work(name: str, delay: float) -> str:
    await asyncio.sleep(delay)
    return name

async def main() -> None:
    start = time.perf_counter()
    async with asyncio.TaskGroup() as group:
        a = group.create_task(io_work("a", 1.0))
        b = group.create_task(io_work("b", 1.0))
    print(a.result(), b.result(), time.perf_counter() - start)

asyncio.run(main())

2 - Event loop, coroutine, task, await

TermDefinition
Coroutine functionDeclared with async def.
Coroutine objectReturned when a coroutine function is called; work has not necessarily run.
TaskA scheduled coroutine managed by the event loop.
AwaitSuspends the current coroutine until an awaitable progresses/completes, allowing other ready tasks to run.
Event loopSchedules ready tasks and monitors I/O readiness/timers.

3 - Choosing the execution model

WorkloadChoiceReason
Thousands of HTTP callsAsync client + semaphore/TaskGroup.High I/O concurrency with controlled in-flight work.
Blocking database driverNormal sync endpoint/thread pool or migrate to async driver.Calling it on the event loop would block all tasks.
Image/PDF CPU parsingProcess pool or worker service.CPU work does not become faster by adding await.
Small maintenance scriptSynchronous code.Async complexity has no payoff.
Mixed API requestAsync orchestration plus offload/blocking boundary.Use each model at the correct resource boundary.

4 - Structured concurrency and failures

Prefer a parent scope that owns child tasks. TaskGroup cancels sibling tasks when one fails and raises grouped errors. gather is still useful when its result ordering and exception behavior are specifically desired.

bounded.py
import asyncio

limit = asyncio.Semaphore(10)

async def fetch(client, url: str) -> str:
    async with limit:
        async with asyncio.timeout(2.0):
            response = await client.get(url)
            response.raise_for_status()
            return response.text

async def fetch_all(client, urls: list[str]) -> list[str]:
    async with asyncio.TaskGroup() as group:
        tasks = [group.create_task(fetch(client, u)) for u in urls]
    return [t.result() for t in tasks]

5 - Blocking, GIL, and offloading

MistakeImpactFix
requests.get() inside async defBlocks the event-loop thread.Use an async HTTP client or asyncio.to_thread as a bridge.
time.sleep() in async codeFreezes unrelated tasks.Use await asyncio.sleep().
CPU loop on event loopHigh tail latency for all requests.Process pool, native/vectorized code, or worker service.
Unbounded create_taskMemory/socket explosion and lost errors.TaskGroup, semaphore, queue, admission control.

6 - Timeouts, cancellation, and backpressure

Cancellation is cooperative and usually appears at await points. Clean up with context managers or finally, do not swallow CancelledError, and propagate a total request deadline. Backpressure means rejecting, queuing, or slowing work before scarce dependencies are saturated.

7 - High-signal interview questions

Q Does async make code faster? ›
It can improve throughput and latency under overlapping I/O waits. It does not make CPU-heavy work execute faster by itself.
Q What happens when you call an async function? ›
It returns a coroutine object. The body progresses when awaited or scheduled on an event loop.
Q AsyncIO vs threads? ›
AsyncIO is cooperative and efficient for async-aware I/O; threads work with blocking libraries but have higher scheduling/memory costs and synchronization concerns.
Q What does the GIL mean here? ›
In normal CPython, one thread executes Python bytecode at a time, so threads do not provide parallel speedup for CPU-bound Python code, though I/O and native extensions can release the GIL.
Q How do you diagnose an event-loop stall? ›
Measure loop lag, inspect slow tasks/traces, look for blocking I/O, time.sleep, CPU loops, synchronous serialization, and overloaded thread pools.

8 - Knowledge check

1 - Which workload best fits a process pool?
Processes enable CPU parallelism and isolation.
2 - Why use a semaphore around outbound requests?
A semaphore creates local backpressure at a scarce boundary.
