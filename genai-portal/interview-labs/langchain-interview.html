LangChain for Interviews

A compact, current, interview-only path through LangChain. Learn what the abstractions mean, where they fit, how they fail, and how to defend your design decisions.

Study + timed coding · 4–6 hDifficulty - IntermediateLangChain - Python - Agents
🎯
Interview outcome

Explain modern LangChain without hiding behind framework jargon, build a typed tool-using agent, distinguish LangChain from LangGraph, and diagnose streaming, tool, state, and observability failures.

1 - What interviewers actually test

Interviewers rarely care whether you memorized every import. They test whether you understand the software boundaries: model interface, messages, tools, structured output, middleware, runtime context, streaming, tracing, and when not to use the framework.

ConceptInterview-ready meaningSignal of depth
Model abstractionA common interface over provider-specific chat models.You still understand provider limits, retries, token accounting, and output differences.
AgentA model-driven loop that selects tools and updates conversation state.You can state authority, stop conditions, and failure handling.
MiddlewareHooks around model/tool execution for policy, logging, routing, and state changes.You know cross-cutting concerns should not leak into every tool.
Runnable/LCELComposable invocation primitives useful for deterministic pipelines.You do not force cyclic agent control flow into a linear chain.
LangGraph boundaryThe lower-level runtime for explicit stateful workflows and durable execution.You choose LangGraph when control flow, persistence, or human pauses must be first class.

2 - Modern LangChain architecture

A strong answer starts with the application boundary, not the framework:

LayerResponsibility
HTTP/service layerAuthentication, request validation, deadlines, rate limits.
Application serviceUse-case orchestration and framework-independent business rules.
LangChain agent or runnableModel calls, tool selection, middleware, structured output.
Tool adaptersTyped wrappers over databases, APIs, queues, and domain services.
Observability/evaluationTraces, tool outcomes, latency, token cost, golden test cases.
minimal_agent.py
from langchain.agents import create_agent
from langchain.tools import tool
from pydantic import BaseModel, Field

class SearchInput(BaseModel):
    query: str = Field(min_length=3, max_length=200)

@tool(args_schema=SearchInput)
async def search_docs(query: str) -> str:
    """Search approved internal documentation."""
    return "bounded result"

agent = create_agent(
    model="provider:model-name",
    tools=[search_docs],
    system_prompt="Use tools for facts. State uncertainty."
)

result = await agent.ainvoke({
    "messages": [{"role": "user", "content": "Find the retry policy"}]
})
🧠
Do not overclaim

A framework can standardize interfaces and orchestration, but it cannot guarantee correct tools, safe authorization, good retrieval, or useful answers. Those remain application responsibilities.

3 - LangChain vs raw SDK vs LangGraph

ChoiceChoose it whenAvoid it when
Raw provider SDKOne or two model calls; exact control; tiny dependency surface.You repeatedly rebuild tools, messages, structured output, tracing, and provider adapters.
LangChainYou need model/tool abstractions, middleware, standard agent setup, integrations, or streaming.A simple call becomes harder to debug than the value the abstraction provides.
LangGraphState, branches, cycles, checkpointing, human approval, durable recovery, or explicit parallel workflow matter.The workflow is a small linear pipeline.

4 - Tools, schemas, structured output

Tool quality is mostly interface design. Use narrow names, explicit descriptions, constrained schemas, bounded output, server-side authorization, and idempotency for side effects. Never expose a generic "run arbitrary SQL" or "call any URL" tool.

tool_boundary.py
from pydantic import BaseModel, Field

class OrderLookup(BaseModel):
    order_id: str = Field(pattern=r"^ORD-[0-9]{6}$")

async def lookup_order(order_id: str, user) -> dict:
    # Schema validation is not authorization.
    if not user.can_read_order(order_id):
        raise PermissionError("not allowed")
    return await repository.get_order(order_id)

5 - Streaming and async

Streaming improves perceived latency and exposes execution progress, but adds cancellation, partial-output, reconnection, and backpressure concerns. Use async APIs for I/O-bound model and tool calls, propagate request cancellation, and limit fan-out.

6 - Failure diagnosis

SymptomLikely causeFirst check
Wrong tool selectedAmbiguous descriptions or overlapping tools.Trace model messages and available tool schemas.
Tool arguments invalidWeak schema or provider output mismatch.Validation errors and raw tool-call payload.
Agent loopsNo stop condition, repeated tool result, or prompt conflict.Step count, repeated state, recursion limit.
Slow first tokenModel/provider latency or pre-model work.Trace spans before streaming begins.
Cannot reproduce issueUnversioned prompt/model/tool set.Persist configuration and trace identifiers.

7 - High-signal interview questions

Q What does LangChain add beyond an SDK? ›
Standard interfaces for models, messages, tools, structured output, middleware, streaming, and integrations. I use it when those abstractions reduce repeated plumbing, not automatically.
Q LangChain vs LangGraph? ›
LangChain is the higher-level application and agent layer; LangGraph is the explicit stateful workflow runtime underneath. I choose LangGraph when state transitions and recovery must be modeled directly.
Q How do you test an agent? ›
Inject deterministic model and tool fakes for software behavior, contract-test schemas, run golden scenario evaluations with real models, and inspect traces for tool choice and stop behavior.
Q How do you prevent framework lock-in? ›
Keep domain logic and service interfaces framework-independent, isolate LangChain in an adapter/orchestration layer, and expose a stable application API.
Q How would you secure tools? ›
Least privilege, narrow schemas, server-side authorization, bounded outputs, idempotency, audit logs, and human approval before consequential writes.

8 - Knowledge check

1 - When is LangGraph a better fit than a simple LangChain runnable?
Explicit stateful control flow and durable pauses are LangGraph concerns.
2 - What is the strongest reason to isolate tools behind service functions?
The agent should call a stable capability boundary, not own domain rules.
