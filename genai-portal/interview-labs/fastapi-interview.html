1 - What FastAPI is built on

FastAPI is an ASGI web framework built on Starlette for web behavior and Pydantic for typed validation/serialization. ASGI supports asynchronous request handling and long-lived connections such as WebSockets.

LayerRole
ASGI server such as UvicornAccepts connections and invokes the application protocol.
StarletteRouting, middleware, requests/responses, WebSockets, background and lifecycle primitives.
FastAPIType-driven routing, dependency injection, OpenAPI integration and application ergonomics.
PydanticInput parsing/validation and output serialization contracts.

2 - Request lifecycle

  1. Connection reaches the ASGI server.
  2. Middleware runs around the request.
  3. Routing chooses the endpoint.
  4. Dependencies resolve and can authorize, open resources, or provide clients.
  5. Parameters/body are validated.
  6. The endpoint calls application services.
  7. The response model serializes output; middleware completes.
app.py
from contextlib import asynccontextmanager
from typing import Annotated
from fastapi import Depends, FastAPI, HTTPException
from pydantic import BaseModel, Field

class ItemCreate(BaseModel):
    name: str = Field(min_length=2, max_length=100)
    quantity: int = Field(ge=1, le=1000)

class ItemView(ItemCreate):
    id: int

@asynccontextmanager
async def lifespan(app: FastAPI):
    app.state.client = await build_client()
    try:
        yield
    finally:
        await app.state.client.aclose()

app = FastAPI(lifespan=lifespan)

async def current_user() -> User:
    user = await authenticate_request()
    if not user:
        raise HTTPException(401, "unauthorized")
    return user

@app.post("/items", response_model=ItemView, status_code=201)
async def create_item(payload: ItemCreate, user: Annotated[User, Depends(current_user)]):
    return await item_service.create(payload, actor=user)

3 - def vs async def

Endpoint styleUse whenRuntime behavior
async defDependencies are awaitable/non-blocking.Runs on the event loop; blocking calls here stall unrelated tasks.
defYou must call a blocking library.FastAPI runs it in an external thread pool and awaits the result.
CPU-heavy workNeither endpoint style should hold it for long.Move to process/worker/background job depending durability needs.
⚠
Common interview trap

Making a function async def does not convert blocking database, file, or HTTP calls into non-blocking operations.

5 - Reliability and security checklist

ConcernInterview anchor
TimeoutsPer-dependency timeout inside a total request deadline.
RetriesOnly transient, safe/idempotent operations; bounded backoff and jitter.
ValidationPydantic validates shape; service layer validates business rules.
Authentication/authorizationValidate issuer/audience/expiry; authorize the resource, not only the route.
Rate limitingPer identity/tenant and endpoint cost; admission before saturation.
ErrorsStable public errors; log internal context with trace ID.
DatabaseConnection pool, transaction scope, indexes, rollback, no session leaks.

6 - Testing strategy

test_app.py
from fastapi.testclient import TestClient


def test_create_item(client: TestClient) -> None:
    response = client.post("/items", json={"name": "pen", "quantity": 2})
    assert response.status_code == 201
    assert response.json()["name"] == "pen"


def test_validation(client: TestClient) -> None:
    response = client.post("/items", json={"name": "", "quantity": 0})
    assert response.status_code == 422

Use unit tests for service logic, integration tests for database/adapters, API tests for routing/validation/auth, and load tests for concurrency, pools, and tail latency. Override dependencies with fakes rather than patching global state everywhere.

7 - High-signal interview questions

Q Why is FastAPI fast? ›
It uses the ASGI ecosystem and Starlette, supports efficient async I/O, and has low framework overhead. Real performance still depends on application code, dependencies, database, serialization, and deployment.
Q What does dependency injection give you? ›
Declarative reusable dependencies for identity, clients, configuration, sessions, and validation, with clean test overrides and OpenAPI integration.
Q Where should business logic live? ›
In services/domain functions, not route handlers or Pydantic validators. Routes translate HTTP; services enforce use cases.
Q BackgroundTasks vs queue? ›
BackgroundTasks is in-process best-effort work after a response. Use a durable queue/workflow system for retries, long work, independent scaling, or crash recovery.
Q How do you deploy it? ›
ASGI server workers behind a proxy/load balancer, health/readiness checks, bounded pools and timeouts, structured logs/metrics/traces, graceful shutdown, and horizontal scaling based on measured workload.

8 - Knowledge check

1 - A blocking SDK must be called in a request. What is safer?
Blocking work must not run on the event-loop thread.
2 - What belongs in a Pydantic validator?
Business permissions and side effects belong outside validation models.
