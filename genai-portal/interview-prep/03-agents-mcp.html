Agents, Tool Use, LangGraph & MCP

The strongest answer distinguishes an agent from a workflow and makes tools, state, authority and stop conditions explicit.

10 high-signal questionsVery current · agentic rolesAnswer aloud, then open
Production agents are controlled loops: bounded tools, explicit state, validation, stop conditions and human escalation.
🎯
How to practise this topic

For every agent answer, state the permitted tools, state model, stopping rule, failure path and human escalation point.

10 questions
01What is an AI agent?agentdefinitionarchitecture＋
Strong answer

An agent is a system in which a model observes state, chooses an action—often a tool call—receives the result and repeats until a goal or stop condition is reached. The production system also includes orchestration, memory, policy, validation and telemetry. A single LLM call that returns JSON is better described as a bounded model task, not automatically an autonomous agent.

30-second version

An agent is a controlled decision loop around a model: observe, decide, act, validate and stop. The surrounding software makes it safe and reliable.

Likely follow-up: Would you call a one-shot resume screener an agent?
02When should you use an agent instead of a fixed workflow?agentworkflowtrade-off＋
Strong answer

Use an agent when the next step genuinely depends on observations and cannot be enumerated economically—for example exploratory research across several tools. Use a workflow when the sequence is known, compliance requires predictability, or the action is high consequence. Many strong systems use a deterministic graph with agentic choice only inside bounded nodes.

30-second version

Use workflows for known paths and agents for dynamic decisions. Keep the agentic surface as small as the problem allows.

Likely follow-up: Give a case where an agent is unnecessary complexity.
03Explain the ReAct pattern.reacttoolsloop＋
Strong answer

ReAct alternates reasoning about the current state with an action such as calling a search or database tool, then uses the observation to decide the next step. In production I implement the loop through structured tool calls rather than parsing free-form text, cap iterations and cost, and validate every action.

30-second version

ReAct is reason, act, observe, repeat. Production versions use structured calls, budgets and policy checks.

Likely follow-up: What are common ReAct failure modes?
04How do you design a good tool for an agent?toolssecurityapi-design＋
Strong answer

A good tool has a narrow verb, typed input, clear description, deterministic authorization and predictable errors. It returns only the data needed, is idempotent when possible, exposes timeouts and never lets model text become code or raw SQL. High-impact tools support dry-run, confirmation or human approval.

30-second version

Make tools small, typed and least-privilege. The server authorizes and executes them; the model only proposes a call.

Likely follow-up: How would you design a tool that sends an email?
05How should multiple agents communicate?multi-agentorchestrationstate＋
Strong answer

I prefer explicit typed messages or shared workflow state over unrestricted natural-language chatter. A supervisor or graph owns routing, each specialist has a narrow role and tool set, and artifacts are stored with provenance. Multi-agent designs need clear termination, conflict resolution, budgets and an explanation of why separate agents outperform simpler modules.

30-second version

Use typed handoffs and an orchestrator, not an unbounded group chat. Separate agents only when roles or permissions truly differ.

Likely follow-up: When does a multi-agent system become counterproductive?
06Why is LangGraph useful for production agents?langgraphstateworkflow＋
Strong answer

LangGraph represents execution as state, nodes and conditional edges, which makes loops, retries, checkpoints, parallel branches and human interrupts explicit. It is useful when an agent must be resumed or audited and when routing is more complex than a simple chain. The framework does not replace good state design or security policy.

30-second version

LangGraph makes agent control flow and durable state explicit, so complex loops and human checkpoints are easier to test and operate.

Likely follow-up: What belongs in graph state, and what should stay outside it?
07What is the Model Context Protocol (MCP)?mcpintegrationsecurity＋
Strong answer

MCP is a protocol for exposing tools, resources and prompt-like capabilities to model applications through a consistent client-server interface. It can reduce custom integration work and make capabilities discoverable, but it does not automatically solve authorization, tenancy or trust. The host must approve servers and tools and validate every invocation.

30-second version

MCP standardizes how AI applications discover and use external capabilities. It improves interoperability, not security by default.

Likely follow-up: How would you secure an enterprise MCP server?
08How do you design memory for an agent?memoryprivacyretrieval＋
Strong answer

I separate short-term execution state, conversation history, user profile and long-term retrieved memory. Each has a different retention, privacy and relevance policy. I summarize or retrieve selectively rather than replaying everything, store provenance, and let users inspect or delete persistent memory.

30-second version

Memory is not one chat transcript. Separate working state from durable, permissioned knowledge and retrieve only what is relevant.

Likely follow-up: How do you prevent stale or incorrect memory from compounding?
09How do you handle tool failures, retries and duplicate actions?reliabilityidempotencytools＋
Strong answer

Classify errors as transient, permanent or policy-related. Retry only transient failures with bounded exponential backoff and jitter, use timeouts and circuit breakers, and attach an idempotency key to side-effecting calls. Persist execution state so a resume does not repeat an already completed action.

30-second version

Retry only safe transient failures, make side effects idempotent, and checkpoint state before and after tools.

Likely follow-up: What if a payment tool times out after the provider accepted it?
10Where should human-in-the-loop controls appear?human-in-loopsafetyux＋
Strong answer

Use a human checkpoint for high-impact, irreversible, low-confidence or policy-sensitive actions. Present the proposed action, evidence, uncertainty and editable fields—not only an approve button. The workflow must handle delays, rejection and escalation, and the decision should be audited.

30-second version

Put humans before consequential actions and give them evidence and control. Design the pause and rejection path as first-class workflow states.

Likely follow-up: How do you avoid approval fatigue?
