10 questions
01Why is async Python useful in a GenAI backend?pythonasyncconcurrency＋
Strong answer

LLM, vector database and tool calls are mostly I/O waits. Async lets one worker serve other requests while awaiting those operations, improving concurrency without a thread per request. It does not make CPU-heavy PDF parsing or local inference faster; those should move to process workers or specialized services.

30-second version

Async improves I/O concurrency for APIs and databases. Move CPU-bound work elsewhere and still bound downstream concurrency.

Likely follow-up: When can async code reduce reliability?
02How would you structure a FastAPI GenAI service?fastapiarchitecturepython＋
Strong answer

Routers handle HTTP validation and authentication, services own business rules and orchestration, repositories own persistence, model adapters hide provider details, and agent or RAG modules expose typed functions. Dependencies inject clients and identity; background work is separated; and every request gets a trace ID and deadline.

30-second version

Keep HTTP, business logic, data access and model integrations separate so each can be tested and replaced.

Likely follow-up: Where should prompt construction live?
03How do you secure a FastAPI application?fastapisecurityrbac＋
Strong answer

Use a trusted identity provider, validate JWT issuer, audience and expiry, enforce authorization in dependencies and services, validate payloads, limit size, configure CORS narrowly, use TLS, protect secrets, rate-limit expensive endpoints and avoid returning internal errors. Database and tool permissions still need least privilege.

30-second version

Authenticate tokens correctly, authorize every resource, validate and limit input, protect secrets and rate-limit costly operations.

Likely follow-up: How do you prevent IDOR between tenants?
04Why is Pydantic valuable in LLM systems?pydanticstructured-outputvalidation＋
Strong answer

Pydantic turns uncertain JSON into explicit typed contracts with required fields, ranges, enums and custom validators. It improves API and model-output reliability and makes tests clearer. Validation is necessary but not sufficient: business rules and authorization still belong in service code.

30-second version

Pydantic catches malformed shape early; services still decide whether the content is permitted and meaningful.

Likely follow-up: How do you handle schema evolution?
05BackgroundTasks, Celery or Temporal—which would you choose?queueworkflowtrade-off＋
Strong answer

FastAPI BackgroundTasks is suitable for small best-effort work tied to one process. Celery is useful for distributed queued jobs with retries. Temporal fits long-running, stateful workflows that need durable timers, retries, compensation and human waits. I choose from durability and orchestration needs, not popularity.

30-second version

Use in-process tasks for trivial best-effort work, a queue for distributed jobs, and durable workflow orchestration for long, stateful processes.

Likely follow-up: Where would an agent workflow benefit from Temporal?
06What is idempotency, and where is it needed?idempotencyretriesdatabase＋
Strong answer

An idempotent operation can be retried without creating a second effect. For emails, payments, interview bookings or workflow transitions, the client supplies or the service derives an idempotency key stored with the result. Retries return the prior outcome, and state transitions use unique constraints or compare-and-set logic.

30-second version

Idempotency makes retries safe. Persist a stable operation key and ensure the side effect happens at most once.

Likely follow-up: How do you handle a timeout with an unknown outcome?
07How do you design multi-tenancy?multi-tenancydatabasesecurity＋
Strong answer

Every domain row carries a tenant identifier, and the authenticated tenant is injected by the server—not accepted as trusted request data. Queries, caches, vector retrieval, object paths and traces are scoped consistently. Higher-risk systems may add row-level security, separate schemas or separate databases.

30-second version

Make tenant scope unavoidable in every storage and cache path, and test cross-tenant access explicitly.

Likely follow-up: When would you use separate databases per tenant?
08What do Docker and Kubernetes contribute to a GenAI stack?dockerkubernetescloud＋
Strong answer

Docker packages runtime dependencies and creates consistent builds. Kubernetes adds scheduling, service discovery, rolling deployment, resource limits and autoscaling, which helps when there are multiple services and variable workloads. It adds operational complexity, so smaller systems may be better on managed containers or serverless platforms.

30-second version

Docker gives repeatable packaging; Kubernetes gives orchestration at a cost. Use only the platform complexity the workload justifies.

Likely follow-up: How would you autoscale an LLM gateway?
09How do you implement timeouts, retries and rate limits?timeoutsretriesrate-limit＋
Strong answer

Propagate a request deadline, set shorter timeouts per dependency, retry only transient and safe operations with bounded exponential backoff plus jitter, and use circuit breakers for unhealthy services. Rate limits should consider tenant, endpoint, token budget and global provider capacity, with clear retry responses.

30-second version

Bound every wait, retry only safe transient failures, and limit admitted work before downstream saturation.

Likely follow-up: Why should retries consume a shared budget?
10How do you test without calling a live model on every test?testingmockingpython＋
Strong answer

Define a provider interface and inject deterministic fakes that return valid, invalid, slow and error responses. Unit-test prompts, schemas, policies and orchestration with fixtures; use recorded or contract tests sparingly; then run a controlled evaluation suite against real models before release.

model_port.py
from typing import Protocol

class Model(Protocol):
    async def generate_json(self, prompt: str) -> dict: ...

class FakeModel:
    def __init__(self, response: dict): self.response = response
    async def generate_json(self, prompt: str) -> dict:
        return self.response
30-second version

Mock the model boundary for software behavior and reserve real calls for contract and quality evaluation.

Likely follow-up: What can a mock never validate?
