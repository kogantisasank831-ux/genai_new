Production Architecture, Latency & Cost

Show that you can turn a model demo into a dependable service with a latency budget, capacity plan and graceful degradation.

10 high-signal questionsHigh frequency · senior rolesAnswer aloud, then open
LLMOps is a feedback loop: test before release, observe after release, then turn failures into new evaluation cases.
🎯
How to practise this topic

Use numbers where possible. Break latency into stages, state the peak load assumption, and explain what degrades when a dependency is slow.

10 questions
01How do you choose a model for a production use case?model-selectiontrade-offcost＋
Strong answer

I begin with task quality and risk, then compare latency, throughput, context length, structured-output reliability, tool support, privacy, region availability and cost. I create a representative eval set and test several model tiers. Often a router sends simple tasks to a smaller model and difficult or high-value tasks to a stronger one, with a safe fallback.

30-second version

Choose from measured task quality under real latency, cost, privacy and reliability constraints—not from leaderboard rank alone.

Likely follow-up: When would you host an open model instead of using an API?
02How do you create a latency budget for a GenAI request?latencysloobservability＋
Strong answer

I split the end-to-end target across gateway, retrieval or tools, prompt assembly, model time-to-first-token, generation, validation and persistence. I measure p50 and p95 per span, run independent I/O in parallel and reserve headroom for retries. The budget is tied to user experience: an interactive chat differs from an offline document job.

30-second version

Break the SLO into measurable spans and optimize the dominant stage. Protect p95, not only the average.

Likely follow-up: What would you do if retrieval is 80 ms but the model takes 8 seconds?
03What does streaming improve, and what does it not improve?streaminguxsafety＋
Strong answer

Streaming reduces perceived latency by showing tokens after time-to-first-token, but it may not reduce total compute or cost. It complicates moderation, structured output, retries and client state because unsafe or invalid content may already be visible. I use it for conversational prose and avoid or buffer it for atomic structured actions.

30-second version

Streaming improves responsiveness, not necessarily total latency. Use it where partial output is useful and safe.

Likely follow-up: How would you stream while still enforcing safety?
04What can be cached in a GenAI application?cachingsecurityperformance＋
Strong answer

Depending on sensitivity and freshness, I can cache embeddings, parsed documents, retrieval results, tool responses, prompt prefixes and final answers for semantically identical requests. Keys must include tenant, permissions, model and data versions. I avoid cross-user leakage, set clear TTLs and invalidate when source data or authorization changes.

30-second version

Cache stable expensive work, but make the key permission-aware and version-aware. Never trade data isolation for speed.

Likely follow-up: How would you implement semantic caching safely?
05How would you handle high concurrency for an LLM-backed API?concurrencyloadasync＋
Strong answer

The API should be async for I/O, with connection pools, per-tenant rate limits, global concurrency limits and a queue for work that need not be interactive. I apply backpressure before the provider limit, cap request size and tokens, isolate heavy workloads and autoscale on queue depth or active generations rather than CPU alone.

30-second version

Use async I/O, bounded concurrency, queues and admission control. LLM capacity—not web-server CPU—is often the real bottleneck.

Likely follow-up: Why can adding more API workers make the system worse?
06When is batching useful?batchingthroughputqueue＋
Strong answer

Batching is useful for offline embeddings, classification or evaluation where a small wait can improve GPU or provider throughput. It is less useful for low-latency interactive generation with different prompt lengths. I group compatible jobs, set maximum batch delay and size, and separate batch queues from user-facing traffic.

30-second version

Batch throughput-oriented work; keep latency-sensitive requests on a controlled online path.

Likely follow-up: How would you avoid one long request slowing a batch?
07How do you reduce token cost without hurting quality?tokenscostoptimization＋
Strong answer

I remove redundant instructions, retrieve fewer but better chunks, compress history, cap output length, route tasks to the smallest model that passes evaluation and avoid repeated context through caching. I track cost per successful task, not cost per request, because a cheap answer that causes retries or manual work is not efficient.

30-second version

Use better context, shorter prompts, model routing and caching—then verify quality. Optimize cost per successful outcome.

Likely follow-up: What token optimization can accidentally reduce quality?
08What is backpressure, and why does an AI service need it?backpressurereliabilityload＋
Strong answer

Backpressure means the system refuses, queues or degrades work when downstream capacity is saturated instead of accepting unlimited requests. LLM APIs, vector stores and tool services have finite concurrency. I use bounded queues, deadlines, 429 or 503 responses with retry guidance, and priority classes so background jobs cannot starve interactive users.

30-second version

Backpressure protects the system by limiting admitted work to downstream capacity. Without it, queues grow until latency and failures cascade.

Likely follow-up: How do you choose between queueing and rejecting?
09How would you deploy a production GenAI service?deploymentclouddocker＋
Strong answer

I containerize a stateless API, keep prompts and release manifests versioned, store durable workflow state externally, and use managed secrets and private networking. Kubernetes or a serverless container platform can autoscale the API; separate workers handle long jobs. I add health checks, canary releases, tracing and an immediate rollback path.

30-second version

Stateless API, external state, managed secrets, separate workers, observable canaries and rollback. Choose the platform from traffic and operational maturity.

Likely follow-up: When is Kubernetes unnecessary?
10Which SLOs would you define?slometricsproduction＋
Strong answer

I define availability, p95 time-to-first-token and total latency, successful task rate, tool and retrieval error rate, safety-event rate and cost per successful task. For agent workflows I also track completion, escalation and loop-limit rates. SLOs should be task- and tier-specific, with graceful degradation documented.

30-second version

Define reliability, latency, task quality, safety and economics. An AI service is healthy only when users complete the task.

Likely follow-up: What should the error budget include?
