Neural Networks, From Zero to Interview-Ready

A focused mental model for candidates who know Python and classical ML but have not yet studied neural networks.

Beginner → intermediateStudy + practice · 12–18 h15 interview questions
🎯
Your interview goal

Be able to explain how a network makes a prediction, how it learns from error, why training fails, and where neural networks appear inside transformers and GenAI systems.

1 · The six ideas to learn

2 · Start with one artificial neuron

A neuron receives numeric inputs, gives each input a learned importance called a weight, adds a bias, then passes the result through an activation function.

z = w₁x₁ + w₂x₂ + … + bthenoutput = activation(z)
A neuron is a small learnable function. Training changes the weights and bias so useful inputs influence the output more.
💡
Connect this to classical ML

Without a non-linear activation, a neuron is essentially a linear or logistic model. Neural networks become powerful by stacking many such transformations and learning intermediate representations automatically.

3 · Layers and the forward pass

A layer applies many neurons in parallel. In a fully connected layer, every output unit sees every input. The dimensions must line up: a layer written as Linear(4, 8) accepts four features and produces eight learned features.

PartWhat it doesInterview language
Input layerReceives features, pixels, embeddings or token vectors.“The feature dimension defines the input width.”
Hidden layersLearn increasingly useful internal representations.“Depth composes simple functions into complex ones.”
Output layerProduces logits, probabilities, a number or an embedding.“The output shape and activation depend on the task.”
🧠
Forward pass

The forward pass is simply the ordered computation from input to prediction using the network’s current parameters. During inference, this is usually all the network does.

Choosing the output for common tasks

Regression

One or more raw numbers. Common loss: mean squared error.

Binary classification

One logit. Common loss: binary cross-entropy with logits.

Multi-class classification

One logit per class. Common loss: cross-entropy.

4 · How the network learns

Training repeats the same loop across batches of examples. This is the highest-value diagram to reproduce on a whiteboard.

Training loop: predict, measure error, propagate gradients backward, update parameters, repeat.

Loss, gradients and backpropagation

  • Loss: the objective minimized during training. It must represent the task and usually be differentiable.
  • Gradient: the direction and sensitivity of loss with respect to a parameter. A large magnitude means a small parameter change strongly affects loss.
  • Backpropagation: efficiently applies the chain rule from the loss back through every operation to compute parameter gradients.
  • Optimizer: uses those gradients to update parameters. AdamW is common for transformer training; SGD remains important and may generalize well in vision tasks.
weight ← weight − learning_rate × gradient

The formula above is basic gradient descent. Real optimizers add momentum, adaptive scaling, weight decay and other controls, but the mental model stays the same.

⚠️
Common interview trap

Do not apply softmax before PyTorch’s CrossEntropyLoss or sigmoid before BCEWithLogitsLoss. These losses accept logits and combine the activation with numerically stable loss computation.

6 · Practical training and debugging

Learning rate

Too high: loss oscillates or becomes NaN. Too low: training is stable but painfully slow or stuck.

Batch size

Larger batches improve throughput and reduce gradient noise but use more memory; they may require learning-rate adjustment.

Initialization

Xavier/Glorot and He initialization keep activation and gradient scales healthy across layers.

Normalization

BatchNorm uses batch statistics; LayerNorm normalizes features per example and is standard in transformers.

Regularization

Weight decay, dropout, data augmentation and early stopping reduce overfitting.

Gradient clipping

Caps gradient norm to control exploding gradients, especially in sequence models.

A disciplined debugging order

  1. Verify data and labels. Inspect a batch, shapes, dtypes, class distribution and preprocessing.
  2. Overfit a tiny batch. A sufficiently expressive model should nearly memorize a few examples. Failure suggests a code, objective or data problem.
  3. Check outputs and loss. Confirm correct output dimensions, logits and target format.
  4. Inspect gradients. Look for None, zeros, huge values or NaNs.
  5. Reduce complexity. Start with a smaller model, lower learning rate and no optional regularization.
  6. Only then tune. Run controlled experiments and track validation metrics, not only training loss.

Train mode versus evaluation mode

In PyTorch, model.train() enables training behavior such as dropout and BatchNorm statistic updates. model.eval() disables dropout and uses stored BatchNorm statistics. Evaluation should also run under torch.no_grad() or torch.inference_mode() to avoid building a gradient graph.

7 · Minimal PyTorch pattern

This is enough code to discuss model structure, logits, loss, backpropagation and optimizer steps in an interview.

binary_classifier.py
import torch
from torch import nn

class BinaryMLP(nn.Module):
    def __init__(self, input_dim: int) -> None:
        super().__init__()
        self.network = nn.Sequential(
            nn.Linear(input_dim, 32),
            nn.ReLU(),
            nn.Dropout(0.2),
            nn.Linear(32, 1),
        )

    def forward(self, x: torch.Tensor) -> torch.Tensor:
        return self.network(x).squeeze(-1)  # logits, not probabilities

model = BinaryMLP(input_dim=20)
optimizer = torch.optim.AdamW(model.parameters(), lr=1e-3)
loss_fn = nn.BCEWithLogitsLoss()

model.train()
for features, labels in train_loader:
    optimizer.zero_grad(set_to_none=True)
    logits = model(features)               # forward pass
    loss = loss_fn(logits, labels.float()) # compare with truth
    loss.backward()                        # backpropagation
    nn.utils.clip_grad_norm_(model.parameters(), max_norm=1.0)
    optimizer.step()                       # update parameters
🔍
Be ready to explain every line

zero_grad prevents gradient accumulation, backward populates gradients, and step changes the parameters. Convert logits to probabilities with torch.sigmoid only for interpretation or thresholding.

8 · How this connects to GenAI

A transformer is not separate from neural networks—it is a neural-network architecture built from embeddings, attention, feed-forward networks, residual connections and normalization.

Neural-network knowledge explains the model; AI engineering adds data, serving, retrieval, evaluation, security and product controls around it.

9 · What an AI engineer should say in interviews

Start with the mental model

“The forward pass produces a prediction. Loss measures error. Backprop computes gradients. The optimizer updates parameters.”

Connect math to operations

Discuss tensor shapes, memory, numerical stability, train/eval mode, batching and monitoring.

Explain failure modes

Overfitting, unstable loss, vanishing gradients, leakage, class imbalance and incorrect objectives.

Show experimental discipline

Use baselines, validation sets, ablations, reproducible seeds, tracked metrics and versioned checkpoints.

10 · Neural network interview questions

15 questions
01What is a neural network, and how is it different from linear regression?fundamentalsrepresentation＋
Strong answer

A neural network composes parameterized linear transformations with non-linear activation functions. Linear regression learns one linear mapping from features to a number; a multi-layer network can learn complex non-linear boundaries and intermediate representations. The trade-off is higher data, compute, tuning and interpretability cost.

30-second version

A neural network is a stack of learned transformations. Non-linear activations let the stack model relationships that one linear equation cannot.

Likely follow-up: What happens if every layer uses only linear operations?
02Explain a forward pass.forwardinference＋
Strong answer

A forward pass applies each layer to the input in order using the current parameters. It produces logits, probabilities, values or embeddings depending on the output head. During training, the framework also records operations needed for automatic differentiation; during inference, gradient tracking is disabled to save memory and compute.

30-second version

It is the input-to-prediction computation. Training keeps the computation graph for gradients; inference does not need it.

Likely follow-up: Why are tensor shapes important in the forward method?
03Why do neural networks need non-linear activation functions?activationlinearity＋
Strong answer

Multiple linear layers without a non-linearity collapse mathematically into one linear transformation, regardless of depth. Activations such as ReLU or GELU let layers compose non-linear functions and learn complex decision boundaries and representations.

30-second version

Without activations, depth adds parameters but not expressive power; the whole network is still linear.

Likely follow-up: Why is ReLU usually preferred over sigmoid in hidden layers?
04What is the difference between a loss and an evaluation metric?lossmetric＋
Strong answer

The loss is the differentiable objective optimized during training. A metric measures task quality in a human-meaningful way and does not have to be differentiable—for example accuracy, F1 or recall. They should align, but the best operational metric may not be suitable as a training loss.

30-second version

Loss drives gradients; metrics judge usefulness. I monitor both training/validation loss and task-specific metrics.

Likely follow-up: Why can validation loss improve while accuracy stays flat?
05Explain backpropagation without deriving every equation.backpropgradients＋
Strong answer

Backpropagation starts from the loss and applies the chain rule backward through the recorded computation graph. It computes the derivative of loss with respect to every trainable parameter, reusing intermediate results efficiently. It does not itself update weights; the optimizer performs the update.

30-second version

Backprop assigns credit or blame to each parameter by propagating loss sensitivity backward. The optimizer then uses those gradients.

Likely follow-up: What does loss.backward() do in PyTorch?
06What does the learning rate control, and how do you recognize a bad value?learning rateoptimization＋
Strong answer

The learning rate controls update scale. If it is too high, loss may oscillate, diverge or become NaN. If it is too low, loss decreases very slowly or appears stuck. I start from established ranges, use a scheduler or warmup when appropriate, and inspect learning curves rather than tuning blindly.

30-second version

It is the optimizer’s step size. Too large is unstable; too small wastes compute and may not reach a useful solution.

Likely follow-up: Why is warmup common in transformer training?
07What are vanishing and exploding gradients?gradientsstability＋
Strong answer

Repeated multiplication through deep networks can make gradients shrink toward zero or grow extremely large. Vanishing gradients stop early layers from learning; exploding gradients cause unstable updates and NaNs. Helpful controls include suitable initialization, non-saturating activations, residual connections, normalization, gradient clipping and appropriate learning rates.

30-second version

The backward signal becomes too small to learn or too large to stay stable. Modern architectures use residuals, normalization and careful initialization to control it.

Likely follow-up: How would you inspect gradient norms in PyTorch?
08How do you choose an activation function?ReLUGELUoutput＋
Strong answer

For hidden layers I usually follow the architecture’s proven default: ReLU-family functions for many MLP/CNN models and GELU or gated variants in transformers. For the output, the task and loss decide: raw logits for numerically stable cross-entropy losses, sigmoid probabilities for binary interpretation, and softmax for a mutually exclusive class distribution.

30-second version

Use a stable non-linearity inside the network; match the output and loss to the task, preferably passing logits directly to combined loss functions.

Likely follow-up: What is a dying ReLU?
09How do you detect and reduce overfitting?overfittingregularization＋
Strong answer

Overfitting appears when training performance keeps improving while validation performance stalls or worsens. I first rule out leakage and distribution mismatch, then use more or better data, augmentation, weight decay, dropout, early stopping, a smaller model or fewer training steps. The validation set must remain separate from tuning decisions.

30-second version

A widening train-validation gap is the signal. Fix data issues first, then reduce effective capacity or add regularization.

Likely follow-up: Why can a very large model still generalize well?
10What is the effect of batch size?batch sizememorythroughput＋
Strong answer

Batch size trades memory, throughput and gradient noise. Larger batches use hardware efficiently and produce smoother gradient estimates but require more memory and may need learning-rate scaling. Smaller batches are noisier, which can help exploration or generalization, but they reduce throughput. Gradient accumulation simulates a larger effective batch when memory is limited.

30-second version

Large batches are efficient but memory-heavy and less noisy; small batches are noisier and slower per epoch. I tune batch size with learning rate and hardware constraints.

Likely follow-up: What is gradient accumulation?
11Compare Batch Normalization and Layer Normalization.BatchNormLayerNormtransformer＋
Strong answer

BatchNorm normalizes each feature using statistics across the mini-batch and maintains running statistics for inference. Its behavior depends on batch composition and size. LayerNorm normalizes across a sample’s feature dimension, so it works consistently for variable-length sequences and small batches; that is why transformers use LayerNorm or RMSNorm.

30-second version

BatchNorm depends on batch statistics; LayerNorm normalizes within each example and is a better fit for sequence models.

Likely follow-up: Why does forgetting model.eval() hurt a BatchNorm model?
12What changes between model.train() and model.eval()?dropoutinferencePyTorch＋
Strong answer

Training mode enables stochastic dropout and makes BatchNorm use and update batch statistics. Evaluation mode disables dropout and uses stored BatchNorm statistics. It does not disable gradients by itself, so inference should also use torch.no_grad() or torch.inference_mode().

30-second version

eval() changes layer behavior; no_grad() changes autograd behavior. Production inference normally needs both.

Likely follow-up: Why might predictions differ between training and evaluation?
13Why does weight initialization matter?initializationXavierHe＋
Strong answer

Initialization sets the scale and symmetry of early activations and gradients. Initializing every weight identically prevents units from learning different features, while scales that are too small or large create vanishing or exploding signals. Xavier initialization is designed for roughly symmetric activations; He initialization is commonly paired with ReLU-family activations.

30-second version

Good initialization keeps signals in a trainable range and breaks symmetry so neurons learn different features.

Likely follow-up: Why are biases often initialized to zero while weights are not?
14Training loss is not decreasing. How do you debug it?debugginglossdata＋
Strong answer

I inspect a raw batch and verify targets, shapes, dtypes and preprocessing; confirm the output head and loss contract; try to overfit a tiny batch; inspect learning rate and gradient norms; remove optional regularization; and compare against a simple baseline. For NaNs I check input ranges, division or logarithm operations, mixed precision, exploding gradients and excessively high learning rate.

30-second version

Start with data and objective correctness, then prove the model can memorize a tiny sample, then inspect gradients and optimization.

Likely follow-up: What would it mean if the model cannot overfit ten examples?
15Where are standard neural-network components inside a transformer?transformerGenAIMLP＋
Strong answer

Token IDs index an embedding matrix; attention uses learned linear projections for queries, keys and values, then softmax weights values; each transformer block contains a feed-forward neural network with an activation such as GELU or SwiGLU; residual connections and normalization stabilize depth; and a final linear layer produces vocabulary logits. Pre-training uses next-token cross-entropy and backpropagation, commonly optimized with AdamW.

30-second version

A transformer is a deep neural network whose distinctive operation is attention. The rest—linear layers, activations, normalization, loss and optimization—comes directly from neural-network fundamentals.

Likely follow-up: What is the role of the feed-forward network after attention?
